# Phase 3: データ保全・復元基盤

## 目的

週案エディタのデータを、誤操作、列変更、AI処理、複数端末での更新、削除操作、年度更新などから守ります。既存のスプレッドシートを自動で並べ替えたり、元データを一括変換したりはしません。

## 保全レイヤー

### 1. 週案の復元ポイント

週案を保存する際、その週の現在値を軽量な復元ポイントとして保存します。自動保存は同一週につき30分間隔で世代化し、短時間の連続編集で履歴が増えすぎないようにします。

- 対象: 行事、登校前、朝学習、1〜6校時、単元、学習内容、休み時間、放課後、宿題、持ち物、振り返り
- 保持期間: 90日
- 自動世代: 同一週につき30分間隔・最大10世代
- 全体上限: 300復元ポイント
- 手動作成にも対応
- 復元前にも現在状態を自動保存するため、復元操作自体を取り消せます

復元を実行する前には、変更される日付と項目をプレビューします。差分を確認してから二段階で確定するため、復元先の選択ミスを防げます。

### 2. スプレッドシート完全バックアップ

その日に初めてアプリを開いた際、スプレッドシート全体のコピーを1件作成します。毎日アプリを使わない日には不要なバックアップを作りません。

- 保持期間: 30日
- 最大: 10件
- 設定画面から手動作成可能
- データベース全消去、学級削除の直前には必ず作成
- バックアップは別スプレッドシートとしてGoogleドライブに作成
- OAuthスコープは既存の `drive.file` のままで、フルDrive権限は追加しません

完全バックアップはスプレッドシート本体のコピーです。学級通信のJSONなど、別Driveファイルへの参照は保持します。学級通信を削除した場合、そのファイルはDriveのごみ箱へ移され、アプリ内のごみ箱から戻すとDrive側も復元を試みます。

### 3. アプリ内ごみ箱

次の削除操作は即時消去せず、30日間ごみ箱へ移動します。

- タスク
- 単元マスタの行
- 保存済み学級通信

設定画面の「データ保全・復元」から復元または完全削除できます。TaskIDが既存タスクと重複する場合は、復元時に新しいIDを付与します。

### 4. 監査ログ

重要な操作を非表示の監査シートへ記録します。

- スキーマ更新
- 週案保存・復元
- 復元ポイント作成
- 完全バックアップ
- ごみ箱への移動・復元・完全削除
- データベース全消去
- 学級削除
- 健全性診断

APIキー、トークン、パスワードなどの名前を持つ項目は `[REDACTED]` に置き換えてから記録します。監査ログは設定画面から閲覧できます。

### 5. 安全なスキーマ更新

保全用の内部シートにはバージョンを持たせ、適用済みの更新を繰り返し実行しません。現在のアプリより新しいスキーマを検出した場合は、推測で書き換えず処理を停止します。

内部シート:

- `_週案_メタ`
- `_週案_監査ログ`
- `_週案_復元ポイント`
- `_週案_ごみ箱`

これらは通常の操作画面を邪魔しないよう非表示になります。削除や列変更はしないでください。

## 列順が異なる既存シートへの対応

Phase 2のヘッダー名ベースの列解決を引き続き使用します。データベース全消去も連続列を前提とせず、見出しから判定した入力列だけを個別にクリアします。振り返りと振り返り状態は全消去の対象外として残します。

## 健全性診断

設定画面から次を確認できます。

- 週案保存に必要な見出しの不足
- 見出しの重複
- 日付の重複
- 日付として認識できない行
- TaskIDの重複

診断は問題を報告するだけで、データを自動修正しません。

## 起動性能への影響

保全画面のJavaScriptは、Phase 2の初期週案描画後に分割モジュールとして遅延取得します。最初の週案表示を待たせず、その後に保全状態と当日バックアップを確認します。

## 緊急時の復旧順序

1. 対象週だけの問題なら「復元ポイント一覧」で差分プレビューを確認してから復元する
2. 削除した項目なら「ごみ箱」から復元する
3. 広範囲の破損なら「バックアップ一覧」から別スプレッドシートを開く
4. 元スプレッドシートへ必要な範囲だけコピーする
5. 列ずれが疑われる場合は「健全性を診断」を実行する

完全バックアップから元スプレッドシートへの自動上書きは行いません。復旧先を間違えて正常データを失う事故を防ぐためです。

## デプロイ

PRをマージした後、Apps ScriptのWebアプリを**新しいバージョンとして再デプロイ**してください。既存URLを維持する場合は、現在のデプロイを編集して新バージョンを指定します。

初回起動時に保全用内部シートが作成され、その日の完全バックアップが実行されます。バックアップ権限の承認画面が出た場合は内容を確認して承認してください。
