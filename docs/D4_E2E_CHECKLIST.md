# D4: 配布前の最終動作チェックリスト（手動E2E）

> **この手順でやること**：デプロイしたあと、実際に人の手で「ひととおり動くか」を確認します。
> 週案アプリは Google Apps Script 上でしか本番動作しないため、**この手動チェックが最終確認**になります。
> **かかる時間の目安**：30〜45分／**用意するもの**：テスト用の Google アカウント2つ（データ分離の確認に必要）

---

## 使い方

- 上から順に、実際に操作しながら ✅ を付けていきます。
- うまくいかない項目があれば、右の「確認ポイント」を見て原因を切り分けます。
- 2アカウント必要な項目（②④）以外は、1アカウントでも確認できます。

---

## ① 新規ユーザーの作成

| 手順 | 期待する結果 |
|------|-------------|
| テスト用アカウントAで共通URLを開く | 初期設定（オンボーディング）画面が出る |
| 「新しく作成する」を選ぶ | 自分専用のデータベースが作られ、週案画面が開く |
| 週案に何か入力して、週を移動して戻る | 入力した内容が保存されている |

- ✅ 新規ユーザーが、自分専用DBを作って使い始められる
- 確認ポイント：作成に失敗する場合は、テンプレート未設定でも動く自動構築（`initializeNewDatabase_`）が働いているか、権限承認済みか。

## ② 別ユーザーとのデータ分離

| 手順 | 期待する結果 |
|------|-------------|
| テスト用アカウントBで同じ共通URLを開く | Aとは別の初期設定画面が出る（Aのデータは見えない） |
| Bで新規作成し、Aと違う内容を入力 | BにはBの内容だけが表示される |
| Aに戻って開き直す | Aの内容のまま（Bの入力は混ざらない） |

- ✅ ユーザーごとにデータが完全に分かれている
- 確認ポイント：混ざる場合、デプロイの「実行＝アクセスしているユーザー」設定を再確認（D1）。

## ③ 既存シートの紐付け

| 手順 | 期待する結果 |
|------|-------------|
| 設定タブ →「使用中のデータベース」→「別のシートに切替」 | URL入力欄が出る |
| 既存スプレッドシートのURLを貼って紐付け | そのシートの内容で週案が表示される |
| 「紐付け解除」→ 確認 | 初期設定画面に戻る（データ自体は消えない） |

- ✅ 既存シートの紐付け・切替・解除ができる

## ④ 個人設定の分離（Gemini・学年など）

| 手順 | 期待する結果 |
|------|-------------|
| Aで担当学年・Gemini APIキーを設定 | Aに保存される |
| Bで設定を見る | Aの設定は見えず、Bは自分で設定できる |

- ✅ 個人設定がユーザーごとに分かれている（UserProperties 化の確認）

## ⑤ Classroom 投稿

| 手順 | 期待する結果 |
|------|-------------|
| 設定タブで連携クラス名を設定し「一覧取得」 | クラス一覧が取得できる（初回は権限承認あり） |
| 「明日の予定を投稿」 | Classroom にお知らせが投稿される |
| （未承認の状態で試す） | 「権限が不足しています。承認してから…」と分かりやすく案内される |

- ✅ Classroom 連携が動く／未承認時は案内が出る（B4 の確認）
- 確認ポイント：Classroom API が有効か（C1）、同意画面にスコープ登録済みか（C2）。

## ⑥ PDF 読み込み（Picker）

| 手順 | 期待する結果 |
|------|-------------|
| 指導計画PDFで「Google Driveから選ぶ」 | Picker が開き、PDFを複数選択できる |
| 選んで「AI解析を開始」 | プレビューが出て、単元マスタに反映できる |
| 行事予定タブで「PDFを追加」→ 学校名を入れて保存 | アプリ専用フォルダに保存され、学校ごとにグループ表示される |

- ✅ Picker からのPDF取り込みと、行事予定の学校別表示が動く（B2 の確認）
- 確認ポイント：Picker が開かない場合、Google Picker API が有効か（C1）。`drive.file` で自分の選んだファイルだけ扱えることを確認。

## ⑦ タスクリマインダー（メール）

| 手順 | 期待する結果 |
|------|-------------|
| タスク画面でリマインダーを有効化・時刻設定 | 設定が保存される（初回は権限承認あり） |
| （未承認の状態で有効化） | 「メール送信の権限が不足しています…」と案内される |
| 設定時刻にメールが届く | 本人宛にリマインダーメールが届く |

- ✅ リマインダーの設定・送信が動く／未承認時は案内が出る

---

## 総合チェック ✅

- [ ] ① 新規ユーザー作成
- [ ] ② 別ユーザーとのデータ分離
- [ ] ③ 既存シートの紐付け・切替・解除
- [ ] ④ 個人設定の分離
- [ ] ⑤ Classroom 投稿
- [ ] ⑥ PDF 読み込み（Picker）・行事予定の学校別表示
