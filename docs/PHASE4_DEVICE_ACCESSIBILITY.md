# Phase 4: 端末最適化・アクセシビリティ

## 目的

週案エディタを、スマートフォン、タブレット、Chromebook、小型ノートPC、デスクトップPCで快適に使用できるようにします。同時に、キーボード操作、スクリーンリーダー、高コントラスト表示、文字拡大、動きの低減へ対応します。

Phase 4は初期週案描画後に読み込まれる独立レイヤーです。Phase 2の高速起動、Phase 3のデータ保全、既存のスプレッドシート構造には影響しません。

## 端末別の表示

### デスクトップ・大型Chromebook

- 従来の上部ナビゲーションを維持
- 週案の曜日ヘッダーと左端の項目名をスクロール中も固定
- タスクサイドバーを従来どおり展開・収納
- 週案を全画面表示するフォーカスモードを利用可能

### タブレット・小型ノートPC

画面幅が1180px以下の場合、今週のタスクを右側のオーバーレイドロワーとして表示します。タスクを開いても週案本体の幅を縮めません。

- ツールバーの「今週のタスク」から開閉
- 背景部分のタップ、閉じるボタン、Escapeキーで閉じる
- 開閉状態を`aria-expanded`で読み上げ支援へ通知

### スマートフォン

画面幅が768px以下の場合、主要機能を下部ナビゲーションへ移します。

主要ナビゲーション:

- 週案
- タスク
- 時数
- 設定
- その他

「その他」には次の機能があります。

- 行事予定
- 単元マスタ
- 学級通信
- PDF読み込み
- 表示テーマ
- 週案の全画面表示

「その他」はダイアログとして動作し、Tabキーのフォーカスがダイアログ外へ抜けないよう制御します。

## スマートフォンの週案表示

初期設定では、週案を1日ずつ画面幅いっぱいに表示します。

- 曜日ボタンで表示日を切り替え
- 週案上を左右にスワイプして前日・翌日へ移動
- 今日の曜日を自動選択
- 選択中セルがある場合は、その曜日を優先
- 土日を非表示にしている場合は曜日切替からも除外

設定画面で「全曜日を横スクロール」を選ぶと、従来の週間表示へ戻せます。

## 週案の全画面表示

画面右下の全画面ボタン、または設定画面からフォーカスモードへ切り替えられます。

フォーカスモードでは次を一時的に隠します。

- アプリヘッダー
- フッター
- タスクサイドバー
- スマートフォン用下部ナビゲーション

週ナビゲーション、編集ツールバー、週案本体は残ります。Escapeキーでも終了できます。

## アクセシビリティ設定

設定画面の「端末表示・アクセシビリティ」で端末ごとに保存します。

### 文字サイズ

- 標準
- 大きい（112%）
- さらに大きい（125%）

### コントラスト

- 標準
- 高コントラスト

高コントラストでは、枠線、補助文字、選択セル、フォーカス表示を強化します。OSの`prefers-contrast`とWindowsの強制カラーモードにも対応します。

### 表示密度

- ゆったり
- コンパクト

コンパクト表示は、画面の高さが限られる端末で1画面に表示できる週案行を増やします。

### アニメーション

- 端末設定に合わせる
- 標準
- 動きを減らす

「端末設定に合わせる」は、OSの`prefers-reduced-motion`を利用します。

### スマートフォンの週案

- 1日ずつ表示
- 全曜日を横スクロール

## スクリーンリーダー対応

週案は見た目のCSS Gridを維持したまま、支援技術には表形式の`grid`として公開します。

- 曜日を`columnheader`として通知
- 行事、登校前、各校時、宿題などを`rowheader`として通知
- 各セルを`gridcell`として通知
- 「曜日、日付、項目、入力内容」の順で読み上げ
- 空欄は「入力なし」と通知
- 祝日名を曜日ヘッダーへ含める
- 選択セルを`aria-selected`で通知
- 編集欄へ「曜日、校時、教科・単元・内容」のラベルを付与

画面上部には「週案の内容へ移動」スキップリンクを設けています。Tabキーで表示されます。

処理完了、エラー、ビュー切替、曜日切替などはライブリージョンで通知します。

## キーボード操作

### 週案セル

| キー | 動作 |
|---|---|
| 矢印キー | 上下左右のセルへ移動 |
| Home / End | 行の先頭日 / 最終日へ移動 |
| Ctrl+Home / Command+Home | 最初のセルへ移動 |
| Ctrl+End / Command+End | 最後のセルへ移動 |
| Enter / F2 | 選択セルを編集 |
| Shift+F10 | 校時セルの操作メニューを開く |
| Space | セルを選択 |

フォーカス対象は常に1セルだけとなるロービング`tabindex`方式です。Tabキーで週案全セルを延々と移動する必要はありません。

### ビュー切替

入力欄以外で次のショートカットを利用できます。

| キー | ビュー |
|---|---|
| Alt+1 | 週案 |
| Alt+2 | タスク |
| Alt+3 | 行事予定 |
| Alt+4 | 時数 |
| Alt+5 | 設定 |

### ツールバーメニュー

- Enter / Space / 下矢印: メニューを開く
- 上下矢印: 項目を移動
- Home / End: 最初 / 最後の項目へ移動
- Escape: メニューを閉じてボタンへ戻る

## タッチ操作

主要ボタン、ナビゲーション、入力欄にはタッチ端末で44px以上の操作領域を確保します。

入力欄は16px以上として、iOS系ブラウザでフォーカス時に意図せず画面が拡大される現象を抑えます。

既存の次の操作も維持します。

- 週案セルのダブルタップで編集
- 校時セルの長押しで操作メニュー
- ピンチズーム
- 2本指スワイプによるビュー切替

1本指の曜日スワイプは、スマートフォンの1日表示時だけ有効です。

## ソフトウェアキーボードと画面回転

`visualViewport`を監視し、スマートフォンのソフトウェアキーボード表示や画面回転時に利用可能な高さを再計算します。

- 下部ナビゲーションが主領域へ重ならないよう`100dvh`とセーフエリアを使用
- 横向きで高さが少ない場合はヘッダーとツールバーを圧縮
- 学級通信のA4ページはスマートフォン幅に合わせて縮小表示
- PDFプレビューとSweetAlert2を画面内へ収める

## フォーカス表示

ボタン、リンク、入力欄、メニュー、週案セルには、マウスクリック時ではなくキーボード操作時に明確なフォーカスリングを表示します。

Windows高コントラストモードではシステムの`Highlight`色を使用します。

## 動的に生成される画面

週案セル、検索結果、AI画面、タスク、ダイアログなど、JavaScriptで後から追加される要素も`MutationObserver`で検出します。

- アイコンを読み上げ対象から除外
- タイトルや表示文字からボタンの読み上げ名を補完
- ラベルのない入力欄に読み上げ名を補完
- 新しいタブで開くリンクを明示
- 独自ダイアログへフォーカストラップを追加

## 起動性能

Phase 4のCSSとJavaScriptは、ページの`load`後350msで取得します。

読み込みに失敗しても既存の週案操作はそのまま利用できます。データ保存API、OAuthスコープ、スプレッドシートには変更を加えません。

## 確認端末マトリクス

デプロイ後、最低限次を確認してください。

| 端末・幅 | 確認内容 |
|---|---|
| 360×800前後のスマートフォン縦 | 下部ナビ、1日表示、曜日スワイプ、入力時の画面拡大防止 |
| スマートフォン横 | 主領域の高さ、ツールバー、下部ナビ |
| 768px前後のタブレット縦 | 1日表示、タスクドロワー、モーダル |
| 1024px前後のタブレット横 | 週間グリッド、タスクドロワー、キーボード |
| Chromebook | ヘッダーナビ、矢印キー、フォーカスリング |
| デスクトップ | 従来レイアウト、全画面週案、ダークモード |
| 画面拡大200% | 横スクロール、文字の欠落、操作可能性 |
| OS高コントラスト | 枠線、選択状態、フォーカス |
| OS動き低減 | アニメーションの停止 |
| スクリーンリーダー | 曜日・項目・セル内容、ビュー切替、処理通知 |

## 推奨スモークテスト

1. スマートフォン幅で週案を開き、今日の曜日だけが表示される
2. 曜日ボタンと左右スワイプで日を切り替えられる
3. 編集モードに入り、入力欄がソフトウェアキーボードに隠れない
4. タブレット幅で今週のタスクを開き、週案幅が縮まない
5. キーボードで週案セルを移動し、Enterで編集できる
6. Shift+F10で校時セルのメニューを開ける
7. Tabキーでスキップリンクを利用できる
8. 文字サイズ、高コントラスト、動きの低減を切り替え、再読込後も維持される
9. 「その他」メニュー内でTabが循環し、Escapeで閉じる
10. 全画面週案を開閉できる
11. ダークモードと高コントラストを併用できる
12. Phase 3のバックアップ・復元画面が従来どおり操作できる

## デプロイ

PRをマージした後、Apps Script Webアプリを新しいバージョンとして再デプロイしてください。

初回表示後にPhase 4資産が遅延読込されます。表示設定は各ブラウザの`localStorage`へ保存されるため、学校共用端末では利用者交代時に「標準に戻す」を実行してください。
