# データベース（DB）スキーマと新規DBの作り方

週案アプリは、各ユーザーのGoogleドライブ上にある **1つのスプレッドシート** を「データベース（DB）」として使用します。
本書は、そのスプレッドシートに必要なシート構成と、新規DBを用意する2つの方法をまとめます。

- 実装参照: `11_Tenant.gs`（`createMyDatabase` / `initializeNewDatabase_` / `DB_CANONICAL_HEADERS`）
- 列の解決: `00_config.gs` の `scanDbHeaderForSheet_()` が **ヘッダー名** で列を解決します（列の並び順は自由ですが、見出し名は下表に一致させてください）。

## 必要なシート

| シート名     | 役割                         | 必須 | 自動生成 |
|--------------|------------------------------|------|----------|
| `データベース` | 週案本体（日付・時程・各校時・振り返り） | ✅   | 新規作成時に構築 |
| `単元マスタ`   | 教科・単元・学習活動のマスタ  | ✅   | 新規作成時に構築 |
| `タスク`       | タスク管理                    | ✅   | `initTaskSheet_()` が遅延生成 |
| `ログ`         | 動作ログ                      | 任意 | `writeToLog_()` が遅延生成 |
| `学級通信`／`学級通信データ` | 学級通信機能             | 任意 | 機能利用時に生成 |

## 「データベース」シートのヘッダー（正規列）

1行目に以下の見出しを置きます（`DB_CANONICAL_HEADERS` と一致）。**並び順は任意**ですが、見出し名はこの通りにしてください。

```
第何週, 日付, 曜日, 時程, 行事, 登校前, 朝学習,
1校時, 単元1, 学習内容1,
2校時, 単元2, 学習内容2,
中休み,
3校時, 単元3, 学習内容3,
4校時, 単元4, 学習内容4,
昼休み,
5校時, 単元5, 学習内容5,
6校時, 単元6, 学習内容6,
放課後, 宿題, 持ち物, 振り返り, 振り返り状態
```

- **必須列**: `日付`・`曜日`・`第何週`（カレンダー生成・週案表示に必須）。
- `日付` 列には日付値（`yyyy/MM/dd`）を、`曜日` 列には `月〜日`、`第何週` 列には週番号を入れます。
- 2行目以降に約1年分（370日）の日付・曜日・週番号を並べます（`initializeNewDatabase_()` は現年度4月始まりで自動生成します。設定タブの「カレンダー再生成」でいつでも作り直せます）。

## 「単元マスタ」シートのヘッダー

```
教科, 単元名, 総時間数, 何時間目, 時間ごとの学習活動
```

## 「タスク」シートのヘッダー

```
TaskID, TaskContent, Resource, DueDate, Source, Status, Priority, Memo
```

## 新規DBを用意する2つの方法

### 方法A: プログラム構築（テンプレート不要・既定）

スクリプトプロパティ `sp_dbTemplateId` が未設定の場合、`createMyDatabase()` は
`SpreadsheetApp.create()` で空のスプレッドシートを作り、`initializeNewDatabase_()` で
上記の `データベース`／`単元マスタ`／`タスク` を自動構築します。配布元の追加設定は不要です。

### 方法B: テンプレート複製

配布元が「検証済みのテンプレート」を用意して配りたい場合:

1. 上記スキーマを満たす完成済みスプレッドシートを1つ用意します。
2. そのファイルを **「リンクを知っている全員が閲覧可（＝複製可）」** に共有します。
3. GASの「プロジェクトの設定 > スクリプト プロパティ」に
   `sp_dbTemplateId = <テンプレートのスプレッドシートID>` を登録します。

`sp_dbTemplateId` が設定されていると、`createMyDatabase()` はテンプレートを `makeCopy()` して
各ユーザーのDBを作成します（方法Aより優先されます）。テンプレートに独自の書式・数式・
サンプル単元などを含めたい場合はこちらを使用してください。

> どちらの方法でも、作成したDBは作成したユーザー本人の所有物になります（開発者はデータを保持しません）。
