# D2: 共通URLを設定して、アプリとして動くか確認する

> **この手順でやること**：D1 で発行した共通URLを `docs/config.js` に書き込み、
> GitHub Pages の「アプリ風の入り口（PWA）」から通しで動くか確認します。
> **かかる時間の目安**：10〜15分／**前提**：D1（共通URLの発行）が終わっていること

---

## はじめに：PWA って？

**PWA** は「ホーム画面に置いて、アプリのように開けるWebページ」のことです。
このリポジトリの `docs/` フォルダが、その入り口（＝PWAシェル）になっています。
ここに共通URLを設定すると、先生はURLを入力しなくてもアプリを開けます。

---

## ステップ1：共通URLを設定する　〔約3分〕

1. `docs/config.js` を開きます。
2. `appUrl: ""` の `""` の中に、D1 でコピーした共通URLを貼り付けます。

   ```js
   window.SCHOOL_PLAN_NOTE_CONFIG = {
     appUrl: "https://script.google.com/macros/s/××××××/exec"
   };
   ```

3. 保存して、GitHub に反映（コミット＆プッシュ）します。

> 💡 **空のままだとどうなる？**
> `appUrl` を空 `""` にしておくと、先生が最初の画面で自分でURLを入力する形になります
> （入力したURLはそのブラウザに保存されます）。共通URLを固定したいなら上のように設定します。

---

## ステップ2：GitHub Pages を公開する　〔約3分〕

1. GitHub リポジトリの **Settings → Pages** を開きます。
2. 公開元（Source）を、`docs/` フォルダを配信する設定にします
   （例：Branch = `main`、Folder = `/docs`）。
3. 数分待つと、`https://（あなたのID）.github.io/SchoolPlan_Editor/` で公開されます。

---

## ステップ3：通しで動作を確認する　〔約7分〕

公開したURLを開いて、次の流れが問題なく進むか確認します。

1. **入り口が開く** → ローディングのあとアプリが表示される
   （`appUrl` 未設定なら、URL入力画面が出る）。
2. **ログイン** → Google のログイン／同意画面が出て、承認できる。
3. **初期設定（オンボーディング）** → 「新規作成」または「既存シートを紐付け」が選べる。
4. **アプリ本体** → 週案の画面が表示され、入力できる。

> ⚠️ **組織（学校）アカウントでは iframe 内に表示できないことがある**
> Google のログインや、Google Workspace（GIGA スクール）などの**組織アカウント**は、
> セキュリティ上アプリ枠（iframe）内に表示されず、Google のエラー画面（「現在、ファイルを開くことができません」＝404）
> が出ることがあります。
> この場合、入り口ページが自動で「アプリを表示できませんでした」という案内に切り替わるので、
> **「アプリを開く」**（同じ画面で開く）または **「新しいタブで開く」** から起動してください。
> 複数の Google アカウントでログインしているときは、**学校のアカウントに切り替えて**からお試しください。

---

## ステップ4：スマホ・PCへの「インストール」を確認　〔約2分〕

- **iPhone / iPad（Safari）**：共有ボタン →「ホーム画面に追加」。
- **Android / PC（Chrome）**：アドレスバーの「インストール」アイコン、または
  画面内の「インストール」ボタンから追加できることを確認します。
- 追加後、ホーム画面のアイコンから起動して、アプリのように全画面で開くか確認します。
