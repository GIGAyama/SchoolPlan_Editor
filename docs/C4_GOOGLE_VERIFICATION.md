# C4: Google の審査（verification）を申し込む

> **この手順でやること**：アプリを「確認済み」にしてもらうため、Google に審査を申し込みます。
> これが通ると、ログイン時の「確認されていないアプリ」警告が消え、だれでも安心して使えます。
> **かかる時間の目安**：申請の準備に30〜60分／**承認まで**：数日〜数週間（Google 側の対応による）
> **前提**：C1・C2・C3 が終わっていること

---

## はじめに：なぜ審査が必要？

Classroom への投稿など「重い機能（sensitive スコープ）」を使うアプリは、
公開前に Google の審査を受けます。審査に通るまでは、初めて使う先生の画面に
**「確認されていないアプリ」** という警告が出ます（テストユーザーは回避可能）。

審査を通せば、この警告が消えます。

> 💡 **朗報**：このアプリは Drive を `drive.file`（ファイル単位）に絞っているので、
> いちばん厳しい審査（restricted スコープ向けの CASA セキュリティ評価）を**避けやすい**作りです。

---

## 申請前の準備チェックリスト

審査をスムーズに進めるため、先に次をそろえます。

### 1. 同意画面まわり（C2・C3 の再確認）
- [ ] アプリ名・サポートメールが正しい
- [ ] プライバシーポリシー／利用規約ページが公開されていて、内容が実際の動きと合っている
- [ ] `{{ }}` プレースホルダが残っていない
- [ ] 承認済みドメインに `github.io` が入っている

### 2. スコープの説明を用意
審査では「なぜその機能が必要か」を聞かれます。次の説明を準備しておきます。

| スコープ | 説明の例 |
|----------|----------|
| `drive.file` | 先生が選んだ／アプリが作ったPDF・出力ファイルだけを扱うため。Drive全体は見ません。 |
| `spreadsheets` | 各先生専用の週案データベース（本人所有）を読み書きするため。 |
| `classroom.courses.readonly` | 連携するクラスの一覧を取得するため。 |
| `classroom.announcements` | 先生が自分のクラスに予定や学級通信を投稿するため。 |
| `script.send_mail` | 先生が設定したタスクのリマインダーを本人宛に送るため。 |
| `userinfo.email` | ログインした先生を見分けて、データを各自に分けるため。 |

### 3. デモ動画を用意（sensitive スコープがある場合）
Classroom の機能を含むので、**操作のデモ動画**が求められます。
- YouTube に「限定公開」でアップし、そのURLを申請に貼ります。
- 内容：ログイン → 同意画面 → 各機能（特に Classroom 投稿）を実際に操作するところ。
- 同意画面に表示されるアプリ名と、動画内のアプリ名を一致させます。

---

## 申請の手順　〔約20分〕

1. Google Cloud の **「OAuth 同意画面」** を開きます。
2. 公開ステータスが「テスト」なら、**「アプリを公開」** で本番環境にします。
3. sensitive スコープがあると、**「確認のため審査を申請」** のような案内が出ます。指示に従って進みます。
4. 求められる情報を入力します。
   - スコープごとの用途説明（上の表を活用）
   - デモ動画のURL
   - プライバシーポリシーのURL
5. 申請を送信します。以降は Google からのメールを待ちます。

---

## 申請したあと

- Google から**追加の質問メール**が来ることがあります。放置すると審査が止まるので、**早めに返信**しましょう。
- 修正を求められたら、同意画面やポリシーを直して再提出します。
- 承認されると「確認されていないアプリ」警告が消えます。

> 📝 **運用メモ**：申請日・担当者・やり取りの記録を残しておくと、
> あとで再申請やスコープ追加のときに役立ちます。
