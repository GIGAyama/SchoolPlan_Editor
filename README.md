# School Plan Note

**School Plan Note** は、小学校（および小中学校）の教員向けに開発された、週案（週間指導計画）および校務支援のWebアプリケーションです。

Google Apps Script (GAS) と Google スプレッドシートをバックエンドとして利用し、最新の生成AI（Gemini API）を組み込むことで、教員の日常的な事務作業の負担を大幅に軽減することを目的としています。

## 🚀 主な機能 (Features)

### 1. デジタル週案（週間指導計画）

*   ブラウザ上の直感的なグリッドUIで、毎日の時程（登校前、朝学習、1〜6校時、放課後など）の教科や学習内容を入力・管理できます。

*   **登校前タスク列**: 「行事」列と「朝学習」列の間に、出勤後・児童登校前に済ませておくタスクを記入できる「登校前」列を用意しています。既存のスプレッドシートに列が無い場合は、カスタムメニュー「週案ツール」>「その他」>「登校前タスク列を追加」で「行事」列の直後に自動追加できます。印刷時は印刷オプションのチェックボックスで表示するかどうかを選べます（初期状態は非表示）。
    
*   **祝日の自動表示**: 内閣府が公開する祝日データを自動取得し、週案グリッドの該当日を赤色で強調して祝日名（例：「海の日」）をバッジ表示します。印刷時も標準・コンパクト両レイアウトで祝日の列を赤系で色分けし、祝日名を表示するため、休みの日が一目で分かります。祝日データはスプレッドシートを開いた際に約1か月ごとに自動更新され、カスタムメニュー「週案ツール」＞「その他」＞「祝日データを更新」からいつでも手動更新できます。

*   **AI自動入力**: 「単元マスタ」の進捗と過去の履歴を照らし合わせ、次の時間の学習内容を自動で割り当てます。

*   **週をまたいだ全文検索**: 週ナビゲーションの「検索」ボタンから、すべての週の入力内容（行事・朝学習・各校時の教科/単元/学習内容・宿題・持ち物・振り返りなど）をキーワードで横断検索できます。「あの実験いつやったっけ」も一発で見つかり、検索結果をクリックするとその週の週案へジャンプします。
    

### 2. 生成AI (Gemini) による業務アシスト

*   **PDF自動読み取り**: 学校配布のPDF（行事予定や指導計画）をAIが解析し、スプレッドシートに転記します。PDFは画面へのドラッグ＆ドロップで直接アップロードできるほか、Google Driveからの選択にも対応。ファイルごとの処理状況が画面に表示され、抽出結果は書き込み前にプレビュー（行事予定は月ごと、単元は教科ごと）で確認・修正・除外してから反映できます。
    
*   **タスク自動抽出**: 週案の予定（実験、特別な行事など）から、事前に準備すべきタスク（準備物、連絡事項など）をAIが自動で洗い出し、優先度付きでリストアップします。
*   **タスク進行管理**: タスクは「未着手→進行中→完了」の3段階で進行を管理でき、優先度（高・中・低）やメモも設定できます。期限切れ・今日が期日のタスクはナビゲーションのバッジや進行状況サマリーで一目で把握でき、毎朝のメールリマインダーで対応漏れを防げます。
    

### 3. 日次振り返り＆AI週報

*   **放課後の自動振り返りモード**: 放課後（15時以降）にアプリを開くと、その日の振り返りモードに自動で遷移。質問（予定との変更点・成果・課題・特記事項）に答えていくだけで、日報形式の振り返りが自動で完成し、データベースの「振り返り」列に記録されます。

*   **音声入力対応**: 振り返りの各入力欄には「音声入力」ボタンが付いており、話すだけで回答を記入できます（Chrome/Edge等の対応ブラウザ）。教室の片付けをしながらでも、その日の振り返りをさっと記録できます。
    
*   **あとで書く（保留）**: その場で対応できない場合は「あとで書く」を選択でき、保留された振り返りには「振り返り」ボタンの一覧から任意のタイミングで取り組めます。
    
*   **週報の自動生成**: その週の授業がある全ての日の振り返りが完了すると、AIが管理職報告用の「週のまとめ」を自動生成し、同一週の日曜日の振り返りセルに保存します。週案の印刷時には、フリースペース（時数表と検印欄の間）に掲載されます。
    

### 4. 時数管理

*   各教科の実施時数を自動集計。学年ごとの標準時数に対する消化割合や進捗をグラフで可視化します。

*   **年度末シミュレーション（過不足予測）**: 「今日までに実施済みの時数」と「明日以降に入力済みの予定時数」から年度末の着地見込みを教科別に計算し、標準時数に対する過不足を予測します。不足が見込まれる教科は警告表示され、「残り必要ペース（時間/週）」も表示されるため、年度途中での指導計画の調整に役立ちます。
    

### 5. 学級通信（ニュースレター）エディタ

*   ブラウザ上で完結するブロックエディタを搭載。見出し、テキスト、画像、来週の時間割、QRコードなどをドラッグ＆ドロップで配置できます。
    
*   作成した通信はPDF化して保存可能です。
    

### 6. Google Classroom 連携

*   **明日の予定の自動投稿**: 毎日指定した時刻に、翌日の時間割・宿題・持ち物をClassroomのストリームへ自動的に投稿します。
    

### 7. 複数学級モード（専科教員・複数学級担当向け）

*   設定タブでONにすると、学級ごとに専用のデータベースシートを持ち、画面左上の切替メニューで学級を切り替えて週案を管理できます。専科教員や複数学級を受け持つ場合に便利です。
    
*   週案・振り返り・時数集計・担当学年・教科別標準時数は**学級ごとに独立**し、タスク・単元マスタ・学級通信・固定時間割・Classroom連携設定は**全学級で共通**です。
    
*   学級を追加すると、既定の「データベース」シートをコピーした専用シート（年間カレンダー付き・入力内容は空）が自動作成されます。モードをOFFに戻しても学級シートとデータは削除されず、既定のシートを使う従来どおりの動作に戻ります。**使わない人の画面には一切表示されない**オプトイン方式です。
    

### 8. 年間行事予定PDFの閲覧（行事予定タブ）

*   学校ごとに作成された年間行事予定のPDFファイルを、アプリ内の「行事予定」タブでそのまま表示・閲覧できます。
    
*   設定画面で指定した「行事予定PDFフォルダ」内のPDFを一覧表示します。フォルダ内に学校名のサブフォルダを作ると学校ごとにグループ表示され、ファイル名・学校名での絞り込み検索もできます。
    
*   「PDFを追加」ボタンから、学校フォルダを指定してPDFを直接アップロードすることも可能です（新しい学校名を入力するとサブフォルダが自動作成されます）。
    

## 🛠 技術スタック (Tech Stack)

*   **Backend**: Google Apps Script (GAS)
    
*   **Database**: Google Spreadsheet
    
*   **Frontend**: HTML5, CSS3, JavaScript (Vanilla JS / SPA Architecture)
    
*   **AI Integration**: Google Gemini API (`gemini-2.5-flash` 等)
    
*   **External API**: Google Classroom API, Google Drive API

## ⚙️ セットアップと導入方法 (Setup & Installation)

このアプリケーションを使用するには、GoogleアカウントとGemini APIキーが必要です。

1.  **プロジェクトのクローン** 本リポジトリをクローンまたはダウンロードします。
    
2.  **Google スプレッドシートの準備** 新規でGoogleスプレッドシートを作成し、「拡張機能」>「Apps Script」を開きます。
    
3.  **ソースコードの配置** GASのエディタ（または `clasp` を使用）に、本リポジトリの `.gs` および `.html` ファイルをすべてコピーして保存します。`clasp` を使う場合は、必要なOAuthスコープ（メール送信 `script.send_mail` などを含む）を宣言した `appsscript.json` も一緒に反映してください。GASエディタで直接編集する場合は「プロジェクトの設定」で「`appsscript.json` マニフェスト ファイルをエディタで表示する」を有効にし、本リポジトリの `appsscript.json` の `oauthScopes` と `dependencies` を反映します。
    
4.  **Google サービスの有効化** GASエディタの「サービス」から以下を追加してください。
    
    *   Google Classroom API
        
    *   Google Drive API
        
5.  **プロパティ（環境変数）の設定** システム設定（APIキー等）を行うため、GASエディタの「プロジェクトの設定」>「スクリプト プロパティ」に以下を追加します。
    
    *   `GEMINI_API_KEY`: 取得したGeminiのAPIキー
        
    *   その他の設定（対象学年やフォルダID）は、Webアプリの設定画面から入力可能です。
        
6.  **Webアプリとしてデプロイ** 「デプロイ」>「新しいデプロイ」を選択し、種類を「ウェブアプリ」にしてデプロイします。発行されたURLにアクセスしてアプリを起動します。
    

> **⚠️ タスクリマインダー（メール通知）を使う場合の再承認について**
> メール送信機能（`MailApp`）を後から追加・更新した場合、既存の承認にはメール送信権限が含まれず、時間ベースのトリガーが「権限不足」で失敗します。上記の `appsscript.json`（`script.send_mail` / `userinfo.email` スコープを含む）を反映したうえで、**アプリを一度開き直して表示される権限リクエストを承認**してください。承認後にWebアプリの「タスク」画面からリマインダーを設定し直すと、正しい権限でトリガーが再作成されます。

> **⚠️ Drive スコープを `drive.file` に最小化しました（再承認が必要）**
> アクセス権を Drive フル（`auth/drive`）から **`auth/drive.file`**（アプリが作成・ユーザーが選択したファイルのみ）へ縮小しました。マニフェスト反映後は**再認可（同意し直し）が必要**です。これに伴う運用変更:
> - **指導計画PDFの取り込み**は、任意フォルダの一覧取得ができなくなったため「**Google Driveから選ぶ**」（Picker で複数選択）に統一しました。
> - **行事予定PDF**は、ユーザー指定フォルダIDではなく**アプリ専用フォルダ（`週案アプリ_行事予定PDF`、初回に自動作成）**に保存されます。学校名を入力するとその配下のサブフォルダに分かれ、「行事予定」タブで学校ごとにグループ表示されます。学校別グループ表示は従来どおり利用できます。
> - 「設定」タブの**「PDF格納フォルダID」は段階的に廃止予定**です（`drive.file` では任意フォルダにアクセスできません）。
> - 週案・学級通信のPDF出力／Classroom 投稿は、アプリが作成するファイルのみを扱うため引き続き動作します。
> - 移行前にフル `drive` スコープで作成した既存フォルダ／ファイルは、`drive.file` 移行後に再認識されない場合があります（行事予定PDFはアプリ専用フォルダへ再アップロードしてください）。詳細は [`docs/B1_DRIVE_SCOPE_AUDIT.md`](docs/B1_DRIVE_SCOPE_AUDIT.md)。

*(※ 詳細なスプレッドシートのシート構成（データベース、単元マスタ、タスク等）は [`docs/DATABASE_SCHEMA.md`](docs/DATABASE_SCHEMA.md) を参照してください。新規作成機能（`createMyDatabase`）はテンプレートが無くても必要シートを自動構築します。)*

## 🏫 複数ユーザーへの配布（マルチテナント・実験的）

1つのWebアプリを共通URLで配布し、**利用者がGoogleログインすると各自のスプレッドシート（DB）に紐付く／新規作成できる**構成に対応しています（`11_Tenant.gs`）。学校をまたいで「誰でも共通URLからすぐ使い始める」運用を想定した仕組みです。

### 配布元の設定手順

1. **スタンドアロンのWebアプリとしてデプロイ**（スプレッドシートにバインドせず、単体のGASプロジェクトとして）。「デプロイ」>「新しいデプロイ」で以下を選びます。
    * **次のユーザーとして実行**：**ウェブアプリケーションにアクセスしているユーザー**
    * **アクセスできるユーザー**：配布範囲に応じて「全員（Googleアカウント）」または「同一ドメインの全員」

2. **DBテンプレートを用意**（任意）。テンプレートを設定しなくても、利用者の「新規作成」はアプリが必要シート（`データベース`・`単元マスタ`・`タスク`）を自動構築するため動作します（`initializeNewDatabase_`）。独自の書式・数式・サンプル単元などを含む「検証済みテンプレート」を配りたい場合のみ、完成済みスプレッドシートを**「リンクを知っている全員が閲覧可（＝複製可）」**に共有し、そのIDをGASの「プロジェクトの設定」>「スクリプト プロパティ」に登録します（設定時はテンプレート複製が優先されます）。詳細は [`docs/DATABASE_SCHEMA.md`](docs/DATABASE_SCHEMA.md)。
    * `sp_dbTemplateId`: テンプレートのスプレッドシートID

3. 利用者は共通URLを開くだけです。初回ログイン時に**オンボーディング画面**が表示され、
    * **新しく作成する**（`sp_dbTemplateId` 設定時のみ）… テンプレートを利用者自身のDriveに複製して開始
    * **既存のシートを紐付け**… 手持ちのスプレッドシートのURL/IDを入力して開始

    のいずれかを選ぶと、以降はその利用者専用のDBが自動で使われます（紐付け先は利用者ごとの `UserProperties` に保存）。

> **後方互換**: 従来どおりスプレッドシートにバインドして使う場合、この画面は表示されず動作は変わりません（バインド先が自動で紐付け済みとして扱われます）。

### 設定の保存先（個人設定 / 共有設定）

マルチテナントで正しく分離できるよう、**個人ごとに変わる設定はユーザー別（`UserProperties`）に保存**します（Gemini APIキー・モデル、担当学年、教科別標準時数、固定時間割、PDFフォルダID、Classroom連携、タスクリマインダー、複数学級設定、初期設定ウィザード完了フラグ 等）。読み取りは「ユーザー別 → 従来のスクリプトプロパティ」の順にフォールバックするため、既存のバインド環境から移行しても設定は失われず、保存し直すと自動的にユーザー別へ移ります（`11_Tenant.gs` の `tGetProp_/tSetProp_`）。

- **共有のまま**: 祝日データ（全国共通）、DBテンプレートID（`sp_dbTemplateId`）など。
- **配布元の注意**: standalone プロジェクトの「スクリプト プロパティ」には個人設定を残さないでください（`sp_dbTemplateId` など共有設定のみ）。全ユーザーへ既定値として見えてしまうためです。
- **PDF一括読み込みのバッチ処理状態もユーザー別（`UserProperties`）**に保存されるため、複数ユーザーが同時にPDF読み込みを実行しても競合しません。バッチ処理はアクセスユーザーとして実行される時間主導トリガーで進むため、状態も同じユーザーの `UserProperties` で整合します。

## 📱 PWA対応（Chromeからアプリとしてインストール）

本アプリはPWA（Progressive Web App）としてインストールできます。ChromeやEdgeのアドレスバーから「アプリをインストール」を選ぶと、独立したウィンドウで動作するデスクトップアプリのように使えます。

> **仕組み**: GASのWebアプリURL（`script.google.com`）はGoogleのラッパーページ経由で配信されるため、そのままではPWAの条件（manifest等）を満たせません。そこで `docs/` フォルダのPWAシェル（manifest + Service Worker）をGitHub Pagesで公開し、その中にGASアプリを全画面のiframeとして埋め込む構成になっています。

### セットアップ手順

1. **GitHub Pages を有効化する**
    本リポジトリ（またはフォーク）の「Settings」>「Pages」を開き、Source を「Deploy from a branch」、Branch を `main` / フォルダを `/docs` に設定して保存します。数分後に `https://<ユーザー名>.github.io/School_plan_note/` が公開されます。

2. **GASアプリのURLを設定する**（どちらか一方でOK）
    *   **画面から設定**: 公開されたページを開くと初回にURL入力画面が表示されるので、GASの「デプロイ」で発行されたWebアプリURL（`https://script.google.com/macros/s/～/exec`）を貼り付けます。URLはそのブラウザのローカルストレージに保存されます。
    *   **ファイルで固定**: `docs/config.js` の `appUrl` にWebアプリURLを記載してコミットすると、全ユーザー共通の接続先として固定されます（学校内で配布する場合はこちらが便利です）。

3. **Chromeでインストールする**
    公開ページをChromeで開き、アドレスバー右端のインストールアイコン（画面のような形のアイコン）をクリックするか、メニュー「キャスト、保存、共有」>「ページをアプリとしてインストール」を選択します。画面左下の⚙メニューにも「アプリとしてインストール」ボタンが表示されます。

### 注意事項

*   GASアプリ側は、デプロイ時の「アクセスできるユーザー」設定に応じてGoogleアカウントへのログインが必要です。Googleのログイン画面はiframe内に表示できないため、画面が表示されない場合は左下の⚙メニューから「新しいタブで開く（ログイン用）」で一度ログインしてから、「再読み込み」してください。
*   Service Workerがキャッシュするのはシェル部分のみです。アプリ本体（GAS）はオンライン環境が必要です。
*   iOS Safariでは「ホーム画面に追加」で同様にアイコンから起動できます。

## 💡 使い方 (Usage)

1.  スプレッドシートを開き、カスタムメニュー「週案ツール」からWebアプリを起動します。
    
2.  初回起動時は、設定が不十分な場合に**初期設定ウィザード**が自動で開き、担当学年 → Gemini APIキー → PDFフォルダ → Classroom連携 → 年間カレンダー生成の順に案内します（「設定」タブの「初期設定ウィザード」ボタンからいつでも再実行できます）。個別に設定する場合は「設定」タブから、PDF保存フォルダのIDやClassroom連携の設定を行ってください。
    
3.  「週案」タブで日々の授業予定を入力します。「AI自動入力」ボタンで単元を自動割り当て可能です。
    
4.  右サイドバーの「タスク」タブを開くと、入力された週案を元にAIが準備物をサジェストします。
