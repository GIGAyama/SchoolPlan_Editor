# Contributing to SchoolPlan Editor

週案エディタは、教員が日常業務で継続利用するデータ中心のアプリです。変更の速さだけでなく、既存データを壊さないこと、学校端末で安定して動くこと、権限を必要以上に広げないことを重視します。

## 開発の基本原則

1. 現在のUI・操作感と後方互換性を維持する。
2. 大きな変更は小さなPRへ分割する。
3. GASの制限、スプレッドシートの既存データ、マルチテナント分離を考慮する。
4. AI生成内容は教師が確認してから適用できる設計を優先する。
5. APIキー、OAuthトークン、児童・保護者情報をリポジトリやIssueへ保存しない。

## ローカル品質チェック

Node.js 20以上を使用します。外部npm依存はなく、初回のみロックファイルを確認してから実行してください。

```bash
npm ci --ignore-scripts
npm run quality
```

`npm run quality`では、次の項目を検証します。

- `.gs`とHTML内JavaScriptの構文
- `App.html`へincludeした状態のJavaScript構文
- include先ファイルの存在
- `appsscript.json`の形式、V8ランタイム、OAuthスコープ
- マージ競合マーカー
- 高確度のAPIキー・秘密鍵パターン
- `postMessage('*')`と`ALLOWALL`の未承認追加
- 品質チェッカー自体の回帰テスト

## ブランチとPR

- ブランチ名は目的が分かる短い名前にします。
- 1つのPRには1つの目的だけを含めます。
- PRテンプレートのリスク、検証、ロールバック方法を記入します。
- UI変更では主要画面幅のスクリーンショットを添付します。
- GAS変更では、静的チェックだけでなく実際のGASデプロイ環境でも確認します。

## GAS変更時の確認

最低限、変更対象に応じて以下を確認します。

- WebアプリURL直接起動とPWA起動
- 新規ユーザーのDB作成・既存DB紐付け
- 週案の読込、編集、保存、週移動、競合時の動作
- 複数学級モードのOFF時互換性と学級切替
- PDF、Gemini、Classroom、Driveなど外部連携の権限エラー
- 権限追加時の`appsscript.json`と再承認手順
- スマートフォン、タブレット、Chromebookでの主要操作

## セキュリティ例外

GASとPWAの制約により、既存の`postMessage('*')`と`XFrameOptionsMode.ALLOWALL`には明示的な例外があります。例外は`quality.config.json`でファイル単位に管理します。新しい例外を追加する場合は、必要性、影響範囲、代替案、緩和策をPRへ記載してください。
