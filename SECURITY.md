# Security Policy

## 脆弱性の報告

児童・保護者情報、学校内資料、APIキー、OAuthトークンなどを公開Issueへ投稿しないでください。脆弱性を発見した場合は、GitHubのPrivate Vulnerability Reportingが有効であれば、リポジトリのSecurityタブから非公開で報告してください。

報告には、影響する機能、再現手順、想定される影響、確認した環境、可能であれば緩和策を含めてください。実在する個人情報や本番APIキーを証拠として添付しないでください。

## 対象

最新の`main`ブランチをセキュリティ修正の対象とします。過去のデプロイは、修正版へ更新してください。

## 機密情報

- Gemini APIキーはスクリプトまたはユーザーのプロパティに保存し、ソースコードへ記載しません。
- OAuthトークンをログ、Issue、スクリーンショットへ残しません。
- スプレッドシートの共有設定とWebアプリの実行ユーザーを変更する場合は、利用者間のデータ分離を再確認します。
